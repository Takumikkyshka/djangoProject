## 3. Выводы
- Основной функционал (просмотр, создание, редактирование) стабилен.
- UX-недочёт после удаления записей требует внимания (Issue UX-002).
- Пользователи просят повысить заметность сообщений об ошибках и подсказок в формах (Issue UX-001, UX-003).
- Дополнительные рекомендации и приоритеты собраны в `issues.md`.

---

## 4. Рекомендации
1. Реализовать заметную кнопку возврата к спискам после удаления.
2. Использовать более контрастные стили для ошибок форм (Bootstrap alert-danger).
3. Добавить подсказки/placeholder для ключевых полей (ISBN, даты) и пометку обязательных полей.
4. Повторно прогнать тест-кейсы после внедрения правок и обновить отчёт.
