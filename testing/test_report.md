## Отчёт о тестировании BookStore

### 1. Функциональное тестирование

Тестирование проводилось вручную в браузерах Google Chrome и Microsoft Edge на локальном сервере разработки.

#### Таблица тест‑кейсов (функциональные сценарии)

| ID  | Сценарий                         | Входные данные                                           | Ожидаемый результат                                                                 | Фактический результат                                               | Статус |
|-----|----------------------------------|----------------------------------------------------------|-------------------------------------------------------------------------------------|----------------------------------------------------------------------|--------|
| F1  | Просмотр списка книг            | Переход на `/books/`                                    | Отображается таблица/список книг без ошибок HTTP                                   | Список книг отображается корректно                                  | Pass   |
| F2  | Просмотр деталей книги          | Клик по книге в списке                                  | Открывается страница с детальной информацией о книге                               | Детальная страница открывается, данные соответствуют записи         | Pass   |
| F3  | Создание книги (валидные данные)| Заполнена форма: заголовок, автор, год, ISBN            | Запись книги создаётся, отображается в списке                                      | Книга создаётся и отображается в списке                             | Pass   |
| F4  | Создание книги (пустые поля)    | Отправка пустой формы                                   | Появляются сообщения об ошибке валидации, запись не создаётся                     | Валидация срабатывает, подсвечиваются обязательные поля             | Pass   |
| F5  | Редактирование книги            | Изменить название существующей книги                     | Название обновляется, в списке и деталях отображается новое значение               | Данные успешно обновляются                                          | Pass   |
| F6  | Удаление книги                  | Подтвердить удаление книги на странице подтверждения     | Запись удаляется, в списке больше не отображается                                  | Книга удалена, в списке отсутствует                                 | Pass   |
| F7  | Просмотр списка авторов         | Переход на `/authors/`                                  | Список авторов отображается без ошибок                                             | Список авторов отображается                                          | Pass   |
| F8  | Создание автора                 | Заполнена форма: имя, фамилия, дата рождения (валидная) | Автор создаётся и отображается в списке                                            | Автор успешно создаётся                                             | Pass   |
| F9  | Доступ в админ‑панель           | Переход на `/admin/`, вход суперпользователем           | Открывается админ‑панель с моделями `Book` и `Author`                              | Админ‑панель доступна, модели видны                                 | Pass   |
| F10 | Ошибочный URL                   | Переход на несуществующий путь, например `/wrong-url/`   | Отображается страница 404 без аварийного падения сервера                           | Отображается стандартная страница 404 Django                        | Pass   |

### 2. Пользовательское тестирование (UX)

Проведено короткое пользовательское тестирование с одногруппниками (3–5 человек).  
После работы с приложением участники заполнили Google‑форму об удобстве интерфейса.

#### Основные сценарии UX‑проверки

| ID  | Сценарий UX                            | Наблюдение / комментарий                                               |
|-----|----------------------------------------|-------------------------------------------------------------------------|
| U1  | Поиск раздела со списком книг         | Пользователи быстро находили раздел «Книги» с главной страницы         |
| U2  | Понимание кнопок создания/редактирования | Кнопки «Добавить книгу» и «Редактировать» были понятны по подписи      |
| U3  | Удобство форм                          | Формы восприняты как понятные, но были замечания по длине некоторых полей |
| U4  | Восприятие сообщений об ошибках        | Сообщения об ошибках форм видны, однако предлагалось сделать их заметнее |
| U5  | Общее впечатление                      | Большинство пользователей оценили интерфейс как «удобный» или «скорее удобный» |

Подробная статистика ответов и выводы — в файле `feedback_summary.md`.
