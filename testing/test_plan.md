## План тестирования BookStore

### 1. Цели тестирования
- **Функциональное тестирование**: проверить корректность основных сценариев работы приложения BookStore:
  - просмотр списка книг и авторов;
  - создание, редактирование и удаление книг и авторов (CRUD);
  - валидация обязательных полей форм.
- **Пользовательское (юнит‑/UX-) тестирование**: оценить удобство интерфейса для конечного пользователя:
  - понятность навигации;
  - очевидность действий (кнопки, ссылки);
  - общее впечатление и предложения по улучшению.

### 2. Объект тестирования
- Веб‑приложение `BookStore` (Django):
  - главная страница (`/`);
  - список и детали книг (`/books/`, `/books/<id>/`);
  - список и детали авторов (`/authors/`, `/authors/<id>/`);
  - формы создания/редактирования/удаления книг и авторов;
  - админ‑панель (`/admin/`) — базовая проверка доступа и отображения моделей.

### 3. Виды тестирования
- **Функциональное тестирование (ручное)**:
  - позитивные сценарии (корректные данные);
  - негативные сценарии (пустые/некорректные поля).
- **Пользовательское тестирование (UX)**:
  - короткая сессия с 3–5 одногруппниками;
  - заполнение Google‑формы с оценкой удобства интерфейса.

### 4. Среда и инструменты
- **Среда**:
  - ОС: Windows 10;
  - Браузеры: Google Chrome (актуальная версия), Microsoft Edge;
  - Сервер разработки: `python manage.py runserver`.
- **Инструменты**:
  - Django (встроенный dev‑сервер);
  - База данных SQLite (файл `db.sqlite3`);
  - Google Forms — для сбора пользовательских отзывов;
  - GitHub — для хранения отчётов.

### 5. Критерии начала и завершения тестирования
- **Начало**:
  - проект развёрнут локально, миграции применены, сервер запускается без ошибок;
  - основные страницы доступны без 5xx‑ошибок.
- **Завершение**:
  - выполнены все запланированные тест‑кейсы (см. `test_report.md`);
  - собраны и проанализированы ответы анкеты (см. `feedback_summary.md`);
  - критические ошибки (блокирующие работу) устранены или заведены в `issues.md`.

### 6. Риски и ограничения
- Небольшое количество пользователей в пользовательском тестировании (одногруппники 3–5 человек).
- Тестирование проводится в учебных целях, покрытие сценариев ограничено основным функционалом.
